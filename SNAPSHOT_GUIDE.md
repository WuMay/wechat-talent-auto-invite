# 按钮截图指南

本指南说明如何截取脚本所需的7个按钮截图。

## 为什么需要截图

脚本通过图像识别来自动点击按钮，相比固定坐标方式更稳定。

## 截图准备

创建 `images` 文件夹，结构如下：

```
your-folder/
├── wechat_talent_invite.ahk
└── images/
    ├── detail_button.png         # 1. 详情按钮
    ├── invite_button.png         # 2. 邀请带货按钮
    ├── add_product_button.png   # 3. 添加商品按钮
    ├── confirm_button.png       # 4. 确认按钮
    ├── send_invite_button.png    # 5. 发送邀约按钮
    ├── confirm_send_button.png  # 6. 确认发送按钮
    └── next_page_button.png     # 7. 下一页按钮
```

## 截图步骤

### 前置准备

1. 打开Chrome浏览器，登录微信小店
2. 进入达人广场页面
3. 使用截图工具（如微信截图、QQ截图等）

### 1. detail_button.png - 详情按钮

**位置**：达人列表中每个达人卡片上的"详情"按钮

**截图要点**：
- 只截取"详情"文字
- 包含按钮边框
- 确保文字清晰
- 建议大小：60x30 像素左右

### 2. invite_button.png - 邀请带货按钮

**位置**：达人详情页右侧的"邀请带货"按钮

**截图要点**：
- 截取完整的按钮文字
- 包含按钮边框
- 确保文字清晰
- 建议大小：100x40 像素左右

### 3. add_product_button.png - 添加商品按钮

**位置**：邀约页面中的"添加上次邀约商品"按钮

**截图要点**：
- 截取"添加"或"添加商品"文字
- 包含按钮边框
- 确保文字清晰
- 建议大小：80x35 像素左右

### 4. confirm_button.png - 确认按钮

**位置**：添加商品后弹窗中的"确认"按钮

**截图要点**：
- 截取"确认"文字
- 包含按钮边框
- 确保文字清晰
- 建议大小：60x30 像素左右

### 5. send_invite_button.png - 发送邀约按钮

**位置**：邀约页面底部的"发送邀约"按钮

**截图要点**：
- 截取"发送邀约"文字
- 包含按钮边框
- 确保文字清晰
- 建议大小：100x40 像素左右

### 6. confirm_send_button.png - 确认发送按钮

**位置**：确认发送弹窗中的"确认"按钮

**截图要点**：
- 截取"确认"文字
- 包含按钮边框
- 确保文字清晰
- 建议大小：60x30 像素左右
- **重要**：这是最终确认按钮，必须截图

### 7. next_page_button.png - 下一页按钮

**位置**：达人广场页面底部的"下一页"或翻页按钮

**截图要点**：
- 截取"下一页"文字或翻页图标
- 包含完整的按钮区域
- 确保文字清晰
- 建议大小：80x30 像素左右

## 截图技巧

### 1. 大小适中
- 只截取按钮的识别部分
- 不要包含太多背景
- 保持清晰度

### 2. 保持原色
- 不要调整颜色
- 保持按钮原始样式

### 3. 文件格式
- 格式：PNG
- 命名：严格按要求命名
- 位置：所有截图放在 `images/` 目录

### 4. 多次尝试
- 如果识别不准确，可以重新截图
- 尝试截取按钮的不同部分
- 可以截图多个版本测试

## 常见问题

**Q: 截图后脚本还是找不到按钮？**
A: 检查以下几点：
1. 截图是否清晰？
2. 截图是否包含了按钮的关键特征？
3. 浏览器缩放是否是100%？
4. 屏幕分辨率是否与截图时一致？

**Q: 可以只截取文字部分吗？**
A: 可以，但建议包含按钮的边框，这样更容易识别。

**Q: 截图需要多精确？**
A: 不需要非常精确，但要包含按钮的主要特征。脚本有一定的容差范围（默认30）。

**Q: 容差值是什么意思？**
A: 容差值越大，识别越宽松（可能会误识别）；容差值越小，识别越严格（可能识别不到）。默认30是合理的值。
